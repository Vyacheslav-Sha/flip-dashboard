# CLIP Analysis Playbook (v1)

Цель: быстро и единообразно оценивать лоты от инвестора (прибыль/риск/ликвидность) и передавать агентам на исполнение.

## 1) Intake (<=5 минут)
Собираем вход:
- Ссылка на лот/торги
- Локация, метро/МЦД, удалённость
- Комнатность, площадь, этаж
- Старт торгов, ожидаемая цена покупки
- Ближайшие аналоги (актив + архив)
- Ограничения/сомнения инвестора

## 2) Экономика флипа (база)
Считаем 3 сценария: pessimistic / base / optimistic.

Формулы:
- `Entry = Purchase + Repair + Transaction + Carry`
- `GrossProfit = Exit - Entry`
- `Margin = GrossProfit / Exit`
- `ROI_cost = GrossProfit / Entry`

Где:
- Repair = площадь * ставка ремонта (по классу отделки)
- Transaction = налоги/комиссии/оформление
- Carry = проценты/обслуживание/простой

## 3) Фильтр ликвидности
Критерии:
- Транспорт (метро/МЦД/выезд)
- Этажность и “штраф этажа”
- Типовой спрос (для старта приоритет: 2к, средний этаж)
- Срок экспозиции аналогов

## 4) Риск-скоринг (0-100)
- Рыночный риск (волатильность/глубина спроса)
- Продуктовый риск (этаж, планировка, локация)
- Исполнительский риск (ремонт, сроки)
- Юридический риск (чистота лота, обременения)

Шкала:
- 0-35: низкий
- 36-60: средний
- 61-100: высокий

## 5) Решение
- `GO`: если base ROI_cost >= 20% и pess не уходит в убыток, риск <= 60
- `WATCH`: если ROI 12-20% или риск 61-70 (нужны допданные)
- `NO-GO`: если ROI < 12% или риск > 70

## 6) Пакет для агентов
Передаём в исполнение:
1. Карточку лота (структурированные поля)
2. Сценарный P&L
3. Список неизвестных с дедлайном валидации
4. Рекомендацию GO/WATCH/NO-GO

## 7) SLA на вход от Дениса
- Первичный скрининг: до 15 минут
- Полный расчёт (с проверкой аналогов): до 60 минут
- Выгрузка в едином формате для принятия решения
