# Contributing

Спасибо за вклад в FLIP Dashboard.

## Как предложить изменение

1. Создайте issue (bug/feature)
2. Сделайте fork и отдельную ветку
3. Внесите изменения
4. Откройте Pull Request

## Рекомендации

- Держите расчётную логику прозрачной и проверяемой
- Не ломайте текущие дефолтные значения модели
- Для визуальных изменений прикладывайте скриншоты «до/после»
- Для формул — кратко объясняйте экономический смысл

## Commit style

Рекомендуемый формат:

- `feat: ...` — новая функция
- `fix: ...` — исправление
- `docs: ...` — документация
- `refactor: ...` — рефакторинг без изменения поведения
